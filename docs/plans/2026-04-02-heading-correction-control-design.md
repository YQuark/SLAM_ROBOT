# 航向纠偏控制设计

日期：2026-04-02

## 目标

把底盘直行控制从“经验补偿优先”收敛到“IMU 航向可信、编码器执行可信”的结构，让上位机拿到的 `yaw_est`、直行姿态和后续建图/导航链路更一致。

## 方案

采用按控制源分层的航向纠偏：

- PC / 上位机命令：使用外环 `yaw_ref - yaw` 生成航向修正量，负责导航直行时长期不跑偏。
- PC / 上位机命令：阻尼项基于 `gz` 的负反馈，负责抑制起步摆头和低附着瞬时偏航。
- PS2 / ESP 人工命令：不使用绝对航向锁定，避免手动转向后被拉回旧航向。
- PS2 / ESP 人工命令：直行时用轻量 `straight_balance` 扶正；IMU 有效时使用车体 Z 轴角速度，IMU 无效时回退编码器 `w_est`。

保持现有四轮 PI 执行层不变。固定侧偏补偿只保留为小幅实车 trim，不作为主要纠偏来源；编码器差速只作为 IMU 无效时的后备输入。

## 关键原则

- 编码器负责轮速执行，不负责证明“车一定走直了”。
- IMU 负责车体朝向和偏航角速度，直行锁航向或扶正优先使用 IMU。
- 人工遥控源优先控制体验，不默认锁定绝对航向。
- 小角度噪声不进入积分，因此给 `yaw_err` 增加小死区。
- 纠偏输出必须限幅，避免低速导航时出现过补偿。

## 预期结果

- 静止时航向漂移维持在较小范围。
- 原地旋转后航向闭合误差维持在几度量级。
- 直行时车头偏转会更快被压住，`yaw_x100` 与真实偏航方向一致。
- PS2/ESP 长按直行时获得扶正，但仍允许用户主动改变方向。
