## 2. 现有功能详细分析

### 2.1 运动控制模块

#### 2.1.1 电机驱动 (motor.c/h)

**实现方式**: 1-PWM + 1-DIR 桥式驱动架构

```
硬件连接原理:
           ┌──────────────┐
MCU PWM ──►│              │
           │  H桥驱动IC   ├──► 电机A
MCU DIR ──►│              │
           └──────────────┘

控制逻辑:
- DIR=0, PWM=duty%  → 正转
- DIR=1, PWM=(1-duty)%  → 反转
- DIR=0, PWM=0  → 空转(Coast)
- DIR=1, PWM=100%  → 刹车(Brake)
```

**关键实现细节**:

| 功能 | 实现方式 | 代码位置 |
|------|----------|----------|
| 速度归一化 | `speed` ∈ [-1.0, +1.0]，自动映射到PWM占空比 | `motor_set()` |
| 占空比限制 | 全局`_duty_limit`，默认0.8，可动态调整 | `motor_set_duty_limit()` |
| 方向极性 | 每电机独立的`dir_polarity`标志，软件翻转 | `motor_set_polarity()` |
| 差速接口 | `v-w`映射到左右轮速度 | `motor_set_diff()` |
| 安全保护 | 零速时强制DIR=0且PWM=0，实现空转 | `motor_set()` |

**核心代码解析**:

```c
void motor_set(motor_id_t id, float speed)
{
    // 1. 输入限幅
    if (speed > 1.0f) speed = 1.0f;
    if (speed < -1.0f) speed = -1.0f;
    
    // 2. 方向极性处理
    speed *= (float)m->dir_polarity;
    
    // 3. 占空比限制
    float d = fabsf(speed);
    if (d > _duty_limit) d = _duty_limit;
    
    // 4. 零速特殊处理 - 空转模式
    if (d < 1e-4f) {
        HAL_GPIO_WritePin(m->dir_port, m->dir_pin, GPIO_PIN_RESET);
        _set_ccr(m->htim, m->ch, 0);
        return;
    }
    
    // 5. 方向设置
    GPIO_PinState dir = (speed >= 0.0f) ? GPIO_PIN_RESET : GPIO_PIN_SET;
    HAL_GPIO_WritePin(m->dir_port, m->dir_pin, dir);
    
    // 6. PWM占空比计算 (DIR=1时需要反转)
    float pwm = (dir == GPIO_PIN_SET) ? (1.0f - d) : d;
    uint32_t ccr = (uint32_t)lroundf(pwm * (float)arr);
    _set_ccr(m->htim, m->ch, ccr);
}
```

**教育级应用价值**:
- 学生可理解PWM调速原理：占空比决定平均电压
- 差速驱动是移动机器人基础算法：v-w坐标系转换
- 方向极性配置展示了软件抽象思想：硬件差异由软件统一

**已有但未完善的保护**:
- ✅ 占空比软件限幅
- ❌ 无硬件过流检测接口
- ❌ 无电机温度监测

#### 2.1.2 编码器模块 (encoder.c/h)

**实现方式**: STM32定时器硬件编码器模式 (Encoder Mode 3)

```
编码器模式原理:
TIMx_SMCR.SMS = 3  (Encoder mode 3: 计数器在TI1和TI2双边沿计数)

        TI1 (A相) ──┬──► 计数器 +1/-1
                     │
        TI2 (B相) ──┴──► 方向判断

分辨率提升: 4倍频 (每周期4个计数)
例如: 11线编码器 → 44 counts/rev
```

**溢出处理算法**:

```c
// 统一用"低16位差分"做溢出处理
// 适用于16位定时器，也适用于32位定时器的低16位
static inline int16_t diff16(uint16_t now16, uint16_t last16)
{
    return (int16_t)(now16 - last16);
}

// 原理: int16_t自动处理环绕
// 例如: now=0x0010, last=0xFFF0
// diff16 = 0x0010 - 0xFFF0 = 0x0020 (正确的+32)
```

**速度估计算法**:

```c
// 一阶IIR低通滤波
// 公式: vel[n] = α × vel[n-1] + (1-α) × raw[n]
// α = ENC_VEL_LPF_ALPHA = 0.80
// 含义: 80%权重给历史值，20%给当前测量

void Encoder_UpdateAll(float dt)
{
    const float a = 0.80f;  // 滤波系数
    const float cps_limit = 1.5f * MAX_CPS;  // 毛刺阈值
    
    for (int i = 0; i < 4; i++) {
        // 1. 读取计数器差分
        uint16_t now16 = read_cnt16(e->htim);
        int16_t d16 = diff16(now16, e->last_cnt);
        e->last_cnt = now16;
        
        // 2. 累积位置
        e->pos += (int32_t)d16;
        
        // 3. 计算原始速度 (counts/s)
        float raw = ((float)d16 * (float)e->polarity) / dt;
        raw -= s_vel_bias_cps[i];  // 减去零偏
        
        // 4. 毛刺保护
        if (fabsf(raw) > cps_limit) {
            raw = 0.0f;  // 异常值丢弃
        }
        
        // 5. 低通滤波
        e->vel_cps = a * e->vel_cps + (1.0f - a) * raw;
    }
}
```

**开机静止校准**:

```c
// 目的: 消除编码器/电路噪声导致的静态零偏
// 方法: 机器人静止时采样800ms，计算平均速度作为零偏

uint8_t Encoder_CalibrateStatic(uint32_t duration_ms, uint32_t sample_interval_ms)
{
    // 采样累积
    while ((HAL_GetTick() - start_ms) < duration_ms) {
        // 每5ms采样一次
        sum_counts[i] += (int32_t)d16 * (int32_t)e->polarity;
    }
    
    // 计算零偏
    float bias = (float)sum_counts[i] / dur_s;
    
    // 阈值检查 (避免把真实运动当作零偏)
    if (fabsf(bias) > 120.0f) {  // ENC_BOOT_CALIB_MAX_BIAS_CPS
        all_ok = 0u;  // 校准期有运动，警告
    } else {
        s_vel_bias_cps[i] = bias;
    }
}
```

**关键参数配置**:

| 参数 | 值 | 说明 |
|------|-----|------|
| MAX_CPS | 13800 | 归一化速度1.0对应的counts/s |
| ENC_VEL_LPF_ALPHA | 0.80 | 滤波系数，越大越平滑 |
| ENC_BOOT_CALIB_MS | 800 | 开机校准时长 |
| ENC_BOOT_CALIB_MAX_BIAS_CPS | 120 | 零偏阈值 |

- 硬件编码器模式是嵌入式经典应用：定时器从模式配置
- 速度滤波展示了信号处理基础：IIR滤波器设计

#### 2.1.3 运动控制核心 (robot_control.c/h)

**控制架构**: 三模式状态机 + 多源指令仲裁 + 4轮独立PI闭环

```
控制模式定义:
┌────────────┬────────────────────────────────────────┐
│   模式     │              功能描述                   │
├────────────┼────────────────────────────────────────┤
│ MODE_IDLE  │ 安全模式，所有输出归零，只观测不控制   │
│MODE_OPEN_LOOP│ 开环测试，直接输出速度命令到电机     │
│MODE_CLOSED_LOOP│ 闭环控制，PI速度环调节电机输出    │
└────────────┴────────────────────────────────────────┘
```

**指令仲裁机制**:

```c
// 设计目标: 
// 1. PS2手柄体验好，不应被心跳包覆盖
// 2. 多源公平，避免固定优先级导致饥饿

static void choose_cmd(uint32_t now_ms, float *v, float *w, cmd_source_t *src)
{
    // 1. 检查各源是否有效 (300ms超时)
    uint8_t ps2_ok = (s_ps2_ts != 0u) && ((now_ms - s_ps2_ts) <= CMD_TIMEOUT_MS);
    uint8_t pc_ok  = (s_pc_ts != 0u) && ((now_ms - s_pc_ts) <= CMD_TIMEOUT_MS);
    uint8_t esp_ok = (s_esp_ts != 0u) && ((now_ms - s_esp_ts) <= CMD_TIMEOUT_MS);
    
    // 2. PS2激活状态最高优先级
    // 原因: PS2摇杆非零时，用户正在操控，不应被心跳零值打断
    if (ps2_ok && (fabsf(s_ps2_v) > 0.03f || fabsf(s_ps2_w) > 0.03f)) {
        *v = s_ps2_v;
        *w = s_ps2_w;
        *src = CMD_SRC_PS2;
        return;
    }
    
    // 3. 其他源按时间戳最新者优先
    uint32_t best_ts = 0u;
    if (ps2_ok && s_ps2_ts >= best_ts) { *v = s_ps2_v; *w = s_ps2_w; best_ts = s_ps2_ts; }
    if (pc_ok && s_pc_ts >= best_ts)   { *v = s_pc_v;  *w = s_pc_w;  best_ts = s_pc_ts; }
    if (esp_ok && s_esp_ts >= best_ts) { *v = s_esp_v; *w = s_esp_w; }
}
```

**速度斜坡限制**:

```c
// 目的: 防止急加速导致电机过流、轮胎打滑、机器人颠簸

static float slew_limit(float current, float target, float rate_per_s, float dt)
{
    float max_step = rate_per_s * dt;  // 本周期最大变化量
    float delta = target - current;
    
    if (delta > max_step)  delta = max_step;   // 限制正向变化
    if (delta < -max_step) delta = -max_step;  // 限制负向变化
    
    return current + delta;
}

// 应用: V_SLEW_MAX = 4.0/s, W_SLEW_MAX = 5.0/s
// 特殊处理: 目标为0时立即归零，不经过斜坡
if (fabsf(v_cmd) < 0.01f) {
    s_v_ref_filt = 0.0f;  // 直接归零，保证刹车响应
} else {
    s_v_ref_filt = slew_limit(s_v_ref_filt, v_cmd, V_SLEW_MAX, dt);
}
```

**PI控制器实现 (带抗积分饱和)**:

```c
// 抗积分饱和 (Anti-Windup) - Back-calculation方法
// 问题: 当输出饱和时，积分项会持续累积，导致超调和响应迟缓
// 解决: 将饱和误差反馈到积分项

typedef struct {
    float kp;        // 比例增益: 0.00025
    float ki;        // 积分增益: 0.00040
    float integral;  // 积分累计
    float out;       // 输出值
} PI_Controller_t;

static float PI_Update_AW(PI_Controller_t *pi, float err, float ff, float dt)
{
    // 1. 计算未饱和输出
    float u_unsat = pi->kp * err + pi->integral + ff;
    
    // 2. 饱和限制
    float u_sat = clampf(u_unsat, -1.0f, 1.0f);
    
    // 3. 抗积分饱和修正
    // 公式: integral += ki*err*dt + Kaw*(u_sat - u_unsat)*dt
    // Kaw = PI_AW_KAW = 0.25
    if (CTRL_USE_AW) {
        pi->integral += pi->ki * err * dt + PI_AW_KAW * (u_sat - u_unsat) * dt;
    } else {
        pi->integral += pi->ki * err * dt;
    }
    
    // 4. 积分限幅
    pi->integral = clampf(pi->integral, -PI_INT_LIM, PI_INT_LIM);  // ±1.0
    
    pi->out = u_sat;
    return u_sat;
}
```

**静摩擦启动辅助**:

```c
// 问题: 低速启动时，静摩擦力>电机驱动力，导致"卡住"
// 解决: 检测到卡住时，注入最小占空比冲破静摩擦

#define START_ASSIST_ENABLE     1u
#define START_ASSIST_MIN_U      0.14f   // 最小辅助占空比 (14%)
#define START_ASSIST_HOLD_MS    80u     // 辅助持续时间

// 检测条件
if (fabsf(ref) > 0.08f * MAX_CPS &&    // 目标速度足够大 (8%满速)
    fabsf(meas) < 0.02f * MAX_CPS) {   // 实际速度接近零 (2%满速)
    // 触发启动辅助
    float assist_u = START_ASSIST_MIN_U * signf_nonzero(ref);
    if (fabsf(u) < START_ASSIST_MIN_U) {
        u = assist_u;             // 注入最小占空比
        pi->integral = 0.0f;      // 清零积分避免累积
    }
}
```

**观测器实现**:

```c
// 速度观测: 基于4轮编码器测量值
static void update_observer(float dt)
{
    // 左右轮平均速度
    float meas_l = 0.5f * (g_encoders[ENC_L1].vel_cps + g_encoders[ENC_L2].vel_cps);
    float meas_r = 0.5f * (g_encoders[ENC_R1].vel_cps + g_encoders[ENC_R2].vel_cps);
    
    // 机器人坐标系速度
    float v_est = 0.5f * (meas_l + meas_r) / MAX_CPS;  // 归一化线速度
    float w_est = 0.5f * (meas_r - meas_l) / MAX_CPS;  // 归一化角速度
    
    s_st.v_est = v_est;
    s_st.w_est = w_est;
    
    // 航向观测: 编码器+IMU融合
    s_yaw_enc += w_est * dt;                           // 编码器积分航向
    if (s_st.imu_valid) {
        s_yaw_imu += (s_imu_last.gz_dps / 180.0f) * dt; // IMU积分航向
    }
    // 加权融合: 70%编码器 + 30%IMU
    s_st.yaw_est = 0.7f * s_yaw_enc + 0.3f * s_yaw_imu;
}
```

#### 2.2.2 电池管理 (battery.c/h)

**实现方式**: ADC电压采样 + 线性电量估算

```
硬件电路:
Vbat (7.2~8.4V) ──┬── 18kΩ ──┬── GND
                  │          │
                  └── ADC ───┘ (0~3.3V)

分压比: (18k + 10k) / 10k = 2.8
ADC电压范围: 0~3.3V → 电池电压范围: 0~9.24V
ADC分辨率: 12bit = 4096级
电压分辨率: 9.24V / 4096 ≈ 2.26mV
```

**电量估算算法**:

```c
// 线性映射 (简化模型)
// 问题: 锂电池放电曲线是非线性的
#define BATT_FULL_V   8.4f   // 满电电压 (2S锂电池串联)
#define BATT_EMPTY_V  6.0f   // 放电终止电压

float voltage_to_percent(float v)
{
    if (v <= BATT_EMPTY_V) return 0.0f;
    if (v >= BATT_FULL_V)  return 100.0f;
    // 线性插值
    return (v - BATT_EMPTY_V) * 100.0f / (BATT_FULL_V - BATT_EMPTY_V);
}

// 实际锂电池放电曲线:
// 100% → 80%: 电压缓慢下降 (4.2V → 3.9V/单体)
// 80% → 20%:  电压平稳 (3.9V → 3.5V/单体)
// 20% → 0%:   电压快速下降 (3.5V → 3.0V/单体)
```

**欠压保护机制**:

```c
// 两级保护策略
#define BATT_LIMIT_VOLTAGE    6.8f   // 限速阈值 (约20%电量)
#define BATT_CUTOFF_VOLTAGE   6.3f   // 停机阈值 (约5%电量)
#define BATT_LIMIT_RECOVER_V  7.0f   // 限速恢复阈值
#define BATT_CUTOFF_RECOVER_V 6.6f   // 停机恢复阈值
#define BATT_LIMIT_DUTY       0.45f  // 限速时的最大占空比

static void Safety_BatteryProtect(void)
{
    const float v = batt.voltage;
    
    // 停机保护 (滞后比较防止抖动)
    if (!s_uv_cutoff_active && v <= BATT_CUTOFF_VOLTAGE) {
        s_uv_cutoff_active = 1;
        RobotControl_SetMode(MODE_IDLE);  // 强制停机
    } else if (s_uv_cutoff_active && v >= BATT_CUTOFF_RECOVER_V) {
        s_uv_cutoff_active = 0;
    }
    
    // 限速保护
    if (!s_uv_limit_active && v <= BATT_LIMIT_VOLTAGE) {
        s_uv_limit_active = 1;
        motor_set_duty_limit(BATT_LIMIT_DUTY);  // 限制最大功率
    } else if (s_uv_limit_active && v >= BATT_LIMIT_RECOVER_V) {
        s_uv_limit_active = 0;
        motor_set_duty_limit(s_nominal_duty_limit);
    }
}
```

**简化模型的局限性**:

| 实际特性 | 当前模型 | 影响 |
|----------|----------|------|
| 非线性放电曲线 | 线性 | 电量估算不准，中间段偏高 |
| 负载压降 | 未考虑 | 负载时显示电量偏低 |
| 温度影响 | 未考虑 | 低温下实际容量下降 |

#### 2.2.3 PS2手柄 (ps2.c/h)

**实现方式**: 软件模拟SPI时序

```c
// DWT精确延时，保证时序稳定
// 优势: 不占用定时器资源，精度高
static void delay_us(uint32_t us)
{
    const uint32_t clk_mhz = SystemCoreClock / 1000000u;  // 168MHz → 168
    const uint32_t start = DWT->CYCCNT;
    while ((DWT->CYCCNT - start) < (us * clk_mhz));
}

// SPI时序 (参考WHEELTEC实现)
// PS2手柄要求: 下降沿采样，时钟频率约250kHz
static uint8_t PS2_RW(uint8_t tx)
{
    uint8_t rx = 0u;
    for (uint8_t ref = 0x01u; ref != 0u; ref <<= 1u) {
        // CMD输出 (MSB first)
        if ((tx & ref) != 0u) PS2_CMD_HIGH();
        else PS2_CMD_LOW();
        
        // 时钟下降沿触发手柄输出
        PS2_CLK_HIGH(); delay_us(5u);
        PS2_CLK_LOW();  delay_us(5u);
        PS2_CLK_HIGH();
        
        // 读取DAT (手柄在下降沿后输出)
        if (PS2_DI()) rx |= ref;
    }
    delay_us(16u);  // 字节间隔
    return rx;
}
```

#### 2.3.1 链路协议 (link_proto.c/h)

**帧结构设计**:

```
┌──────┬──────┬──────┬───────┬──────┬──────────┬──────────┬───────┬──────┐
│ SOF  │ VER  │ TYPE │ FLAGS │ SEQ  │ LEN(2B)  │ PAYLOAD  │ CRC16 │ EOF  │
│ 0xA5 │ 0x01 │ cmd  │ flags │ seq  │ len      │ data...  │ crc   │ 0x5A │
└──────┴──────┴──────┴───────┴──────┴──────────┴──────────┴───────┴──────┘
  1B     1B     1B      1B      1B      2B         nB         2B      1B

帧长度 = 10 + payload_len (字节)
最小帧长度: 10字节 (无payload)
最大payload: 192字节
```

**COBS编码原理**:

```c
// 目的: 消除帧内的0x00，使0x00可作为帧分隔符
// COBS = Consistent Overhead Byte Stuffing

// 编码原理: 用非零字节记录下一个0x00的位置
// 例如: 
//   原始: [0x01][0x02][0x00][0x03][0x04]
//   编码: [0x03][0x01][0x02][0x03][0x03][0x04][0x00]
//          ↑      数据    ↑  ↑  数据   ↑  ↑
//          |              |  |         |  |
//    距离下个0是3字节      |  距离下个0是3字节
//                          这里的0被编码为距离

static uint16_t cobs_encode(const uint8_t *input, uint16_t len, 
                            uint8_t *output, uint16_t out_max)
{
    uint16_t read_index = 0;
    uint16_t write_index = 1;
    uint16_t code_index = 0;
    uint8_t code = 1;
    
    while (read_index < len) {
        if (input[read_index] == 0x00) {
            output[code_index] = code;  // 写入距离
            code = 1;
            code_index = write_index++;
            read_index++;
        } else {
            output[write_index++] = input[read_index++];
            code++;
        }
    }
    output[code_index] = code;
    return write_index;
}
```

**CRC校验**:

```c
// CRC16-CCITT-FALSE
// 多项式: 0x1021 (x^16 + x^12 + x^5 + 1)
// 初始值: 0xFFFF
// 无输入反转，无输出反转
static uint16_t crc16_ccitt_false(const uint8_t *data, uint16_t len)
{
    uint16_t crc = 0xFFFFu;
    for (uint16_t i = 0; i < len; ++i) {
        crc ^= (uint16_t)data[i] << 8;
        for (uint8_t j = 0; j < 8; ++j) {
            if (crc & 0x8000u) 
                crc = (crc << 1) ^ 0x1021u;
            else 
                crc <<= 1;
        }
    }
    return crc;
}
```

- ROS2无法获取可靠的IMU数据
- 建图时里程计漂移无法校正

**需要实现的功能**:

```
输入: 
  - ax, ay, az (加速度计, 单位:g)
  - gx, gy, gz (陀螺仪, 单位:°/s)

处理流程:
  1. 零偏校准 (静止N秒采样，计算均值)
  2. 姿态融合 (互补滤波/Mahony)
  3. 坐标变换 (机体坐标系→世界坐标系)

输出:
  - roll, pitch, yaw (欧拉角, 单位:°)
  - qw, qx, qy, qz (四元数)
  - linear_ax, linear_ay, linear_az (线性加速度，去除重力)
```

**推荐算法**: 互补滤波 (代码量小，适合教学)

```c
// 简化互补滤波算法
// 原理: 加速度计低频可靠，陀螺仪高频可靠，加权融合

#define GYRO_WEIGHT  0.98f  // 陀螺仪权重 (越大越依赖陀螺仪)
#define DT           0.01f  // 采样周期 (100Hz)

typedef struct {
    float roll, pitch, yaw;     // 欧拉角 (°)
    float qw, qx, qy, qz;       // 四元数
    float gyro_bias_x, gyro_bias_y, gyro_bias_z;  // 零偏
} Attitude_t;

void Attitude_Update(Attitude_t *att, 
                     float ax, float ay, float az,
                     float gx, float gy, float gz, float dt)
{
    // 1. 减去零偏
    gx -= att->gyro_bias_x;
    gy -= att->gyro_bias_y;
    gz -= att->gyro_bias_z;
    
    // 2. 加速度计计算倾斜角 (当物体静止或匀速时可靠)
    float acc_roll = atan2f(ay, az) * 57.3f;   // rad → °
    float acc_pitch = atan2f(-ax, sqrtf(ay*ay + az*az)) * 57.3f;
    
    // 3. 陀螺仪积分 (短期可靠)
    att->roll += gx * dt;
    att->pitch += gy * dt;
    att->yaw += gz * dt;  // yaw没有加速度参考，只能靠陀螺仪
    
    // 4. 互补滤波融合
    // roll/pitch: 陀螺仪积分 + 加速度计修正
    att->roll = GYRO_WEIGHT * att->roll + (1.0f - GYRO_WEIGHT) * acc_roll;
    att->pitch = GYRO_WEIGHT * att->pitch + (1.0f - GYRO_WEIGHT) * acc_pitch;
    // yaw: 仅陀螺仪 (需要磁力计修正)
    
    // 5. 角度归一化
    if (att->yaw > 180.0f) att->yaw -= 360.0f;
    if (att->yaw < -180.0f) att->yaw += 360.0f;
}

// 零偏校准 (静止时采样)
void Attitude_Calibrate(Attitude_t *att, float gx, float gy, float gz)
{
    static float sum_x = 0, sum_y = 0, sum_z = 0;
    static int count = 0;
    
    sum_x += gx; sum_y += gy; sum_z += gz;
    count++;
    
    if (count >= 1000) {  // 10秒 @ 100Hz
        att->gyro_bias_x = sum_x / count;
        att->gyro_bias_y = sum_y / count;
        att->gyro_bias_z = sum_z / count;
        count = 0; sum_x = sum_y = sum_z = 0;
    }
}
```

#### 3.1.3 控制算法完善 (优先级: P1)

**问题**: 双环控制和航向锁定未启用

**当前配置**:

```c
#define CTRL_USE_DUAL_LOOP  0u  // 禁用
#define CTRL_USE_YAW_HOLD   0u  // 禁用
```

**双环控制原理**:

```
单环控制 (当前):
  v_cmd, w_cmd → 轮速分配 → PI控制器 → PWM输出
  
双环控制 (待启用):
  外环 (速度环): 
    v_ref, w_ref → v_err, w_err → PI控制器 → v_cmd, w_cmd
  内环 (轮速环): 
    v_cmd, w_cmd → 轮速分配 → PI控制器 → PWM输出

优势:
  - 外环补偿模型误差 (轮距、轮径误差)
  - 更好的直线性和轨迹跟踪
  - 对负载变化更鲁棒

外环参数:
  OUTER_V_KP = 0.60
  OUTER_V_KI = 0.20
  OUTER_W_KP = 0.70
  OUTER_W_KI = 0.22
  OUTER_INT_LIM = 0.40
```

**航向锁定原理**:

```c
// 目的: 直行时自动修正航向偏差，减少侧向漂移

#define YAW_HOLD_KP        1.10f   // 航向修正增益
#define YAW_HOLD_W_THRESH  0.06f   // 角速度阈值，低于此值认为是直行

void yaw_hold_control(float *w_cmd, float yaw_ref, float yaw_est)
{
    // 仅在直行时启用
    if (fabsf(*w_cmd) >= YAW_HOLD_W_THRESH) {
        s_yaw_hold_ref = yaw_est;  // 更新参考航向
        return;
    }
    
    // 计算航向误差 (处理±180°边界)
    float yaw_err = yaw_ref - yaw_est;
    if (yaw_err > 180.0f) yaw_err -= 360.0f;
    if (yaw_err < -180.0f) yaw_err += 360.0f;
    
    // 修正角速度
    float w_corr = YAW_HOLD_KP * yaw_err * 3.14159f / 180.0f;  // ° → rad
    w_corr = clampf(w_corr, -0.3f, 0.3f);  // 限幅
    
    *w_cmd += w_corr;
}
```

```
  2. 滤波: 移动平均 (N=8)

  4. 保护动作: 降功率 + 警告 + 日志
```

#### 3.2.2 编码器断线检测

**现状**: 无编码器信号质量监控

**影响**: 编码器断线时闭环控制失控，可能导致机器人突然加速

**建议实现**:

```c
typedef struct {
    uint32_t stuck_ticks;      // 卡住计数
    float prev_pwm;            // 上次PWM
} encoder_health_t;

encoder_health_t enc_health[4];

void check_encoder_health(encoder_health_t *h, float pwm, float vel)
{
    // 检测条件: 有PWM输出但编码器速度持续为零
    if (fabsf(pwm) > 0.1f && fabsf(vel) < 10.0f) {
        if (++h->stuck_ticks > 100) {  // 2秒 @ 50Hz
            // 触发编码器故障
            trigger_fault(FAULT_ENCODER_STUCK);
            RobotControl_SetMode(MODE_IDLE);  // 安全停机
        }
    } else {
        h->stuck_ticks = 0;
    }
}
```
